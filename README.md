## 📋 Обзор

**Remnawave Monitor** — self-hosted веб-панель для мониторинга и защиты вашей [Remnawave](https://github.com/remnawave/backend) VPN-панели от злоупотреблений.

Система автоматически анализирует подключения пользователей, обнаруживает передачу подписки третьим лицам и помогает оператору принимать решения — вручную или с помощью ИИ.

Без внешних фреймворков. Чистый Node.js + vanilla JavaScript. Единственная зависимость: `better-sqlite3`.

- Смягчающие факторы — автоматические пояснения, почему сигнал может быть ложным
- Учёт мобильных пользователей за CGNAT — минимум ложных срабатываний

### 🚨 Управление инцидентами
- Статус-машина: `Новый → В работе → Предупреждён → Закрыт / Забанен`
- Заметки оператора и журнал действий
- Автоматическое переоткрытие при повторной детекции
- Отправка предупреждений через Telegram

### 🔗 Граф связей
- Анализ кластеров: общие IP, общие провайдеры, общие устройства
- Обнаружение мультиаккаунтов через совпадение устройств
- Скользящее окно 30 минут

### ⚙️ Движок правил
- Настраиваемые правила с условиями и порогами
- Поддержка операторов: >, >=, <, <=, ==, !=, is_true, is_false

## 🚀 Быстрый старт

### Требования

- **Node.js** ≥ 18.0.0
- Работающая панель **Remnawave** с доступом к API
- *(Опционально)* Инструменты сборки для компиляции `better-sqlite3`: `build-essential`, `python3`

### Автоматическая установка (рекомендуется)

Интерактивный установщик проведёт вас через весь процесс:

```bash
git clone https://github.com/Nurmaga095/remnawave-monitor.git
cd remnawave-monitor
sudo chmod +x setup.sh
sudo ./setup.sh
```

Установщик выполнит:
- ✅ Проверку и установку Node.js при необходимости
- ✅ Установку инструментов сборки для `better-sqlite3`
- ✅ Запрос всех необходимых настроек (логин, пароль, URL панели, API-токен)
- ✅ Генерацию безопасного секрета сессии
- ✅ Создание `.env` с правильными правами доступа
- ✅ Установку npm-зависимостей и проверку синтаксиса всех файлов
- ✅ Создание системного пользователя
- ✅ Настройку systemd-сервиса (автозапуск при загрузке)
- ✅ Опциональную настройку Caddy reverse proxy с HTTPS

### Ручная установка

```bash
git clone https://github.com/Nurmaga095/remnawave-monitor.git
cd remnawave-monitor
cp .env.example .env
npm install --omit=dev
```

### Настройка

Отредактируйте `.env`:

```env
# Учётные данные для входа в панель
APP_USERNAME=admin
APP_PASSWORD=ваш-надёжный-пароль
SESSION_SECRET=замените-на-минимум-32-случайных-символа

# Подключение к Remnawave
REMNAWAVE_BASE_URL=https://your-panel.example.com
REMNAWAVE_API_TOKEN=ваш-api-токен-remnawave

# Опционально: Telegram-предупреждения
TELEGRAM_BOT_TOKEN=токен-вашего-бота
TELEGRAM_CHAT_ID=-1001234567890
TELEGRAM_TOPIC_ID=12345
```

### Запуск

```bash
npm start
```

Откройте `http://127.0.0.1:8787` и войдите с вашими учётными данными.

## 🌐 Деплой на VPS

### Ubuntu / Debian

#### 1. Установка зависимостей

```bash
# Node.js 20+
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Инструменты сборки (для better-sqlite3)
sudo apt install -y build-essential python3 make g++

# Caddy (reverse proxy)
sudo apt install -y caddy
```

#### 2. Деплой приложения

```bash
sudo git clone https://github.com/Nurmaga095/remnawave-monitor.git /opt/remnawave-monitor
cd /opt/remnawave-monitor
sudo cp .env.example .env
sudo nano .env  # заполните ваши значения
sudo npm install --omit=dev
```

#### 3. Создание системного пользователя

```bash
sudo useradd --system --home /opt/remnawave-monitor --shell /usr/sbin/nologin remnawave
sudo chown -R remnawave:remnawave /opt/remnawave-monitor
```

#### 4. Установка systemd-сервиса

```bash
sudo cp /opt/remnawave-monitor/remnawave-monitor.service.example \
        /etc/systemd/system/remnawave-monitor.service
sudo systemctl daemon-reload
sudo systemctl enable --now remnawave-monitor
```

#### 5. Настройка Caddy

```bash
sudo cp /opt/remnawave-monitor/Caddyfile.example /etc/caddy/Caddyfile
sudo nano /etc/caddy/Caddyfile  # замените your-domain.example на ваш домен
sudo systemctl reload caddy
```

#### 6. Файрвол

Откройте только порты `22`, `80`, `443`. Порт `8787` **не** должен быть открыт наружу — Caddy проксирует его через HTTPS.

## 🔄 Обновление

Когда в репозитории появляется новая версия, обновите установку на сервере:

```bash
cd /opt/remnawave-monitor
sudo systemctl stop remnawave-monitor
sudo git pull origin main
sudo npm install --omit=dev
sudo systemctl start remnawave-monitor
```

> **Примечание:** файлы `.env` и `data/` (база данных) не хранятся в репозитории и не будут затронуты при обновлении. Все ваши настройки и данные сохранятся.

Если вы хотите проверить, что обновление прошло успешно:

```bash
sudo systemctl status remnawave-monitor
sudo journalctl -u remnawave-monitor -n 30 --no-pager
```

## 🔍 Движок детекции v6

Движок анализирует поведение пользователей и обнаруживает передачу подписки третьим лицам. Работает автоматически после каждого цикла синхронизации.

### Уровни риска

| Баллы | Уровень | Описание |
|---|---|---|
| 60–100 | 🔴 Критический | Устройств больше лимита тарифа |
| 40–59 | 🟠 Высокий | 2+ сильных сигнала из разных категорий |
| 20–39 | 🟡 Внимание | 1 сильный или 3+ слабых сигнала |
| 0–19 | 🟢 Чисто | Нет значимых признаков |

### Сигналы обнаружения

#### Устройства
| Сигнал | Описание |
|---|---|
| Устройств больше лимита | Зарегистрировано устройств больше, чем разрешено тарифом |
| Частая смена устройств | За 30 дней сменено слишком много устройств |
| Общее устройство | Одно устройство найдено в нескольких аккаунтах |

#### Геолокация
| Сигнал | Описание |
|---|---|
| Невозможное перемещение | Скорость между IP > 900 км/ч |
| Одновременно из далёких городов | Подключения из 2-3+ городов одновременно |

#### Сети и провайдеры
| Сигнал | Описание |
|---|---|
| Разные провайдеры одновременно | Число провайдеров превышает лимит устройств |
| Извлечение ключа | Устройство одно, но провайдеров больше лимита |
| Несколько серверов | Одновременно на нескольких узлах сверх лимита |
| VPN/Proxy/Дата-центр | Подключения через VPN, прокси или хостинг |

#### Поведение
| Сигнал | Описание |
|---|---|
| Активность 24/7 | Круглосуточная активность — похоже на нескольких человек |
| Резкое изменение | Резкий рост часов активности |
| Расписание | Разные провайдеры утром и вечером |

#### Подписки
| Сигнал | Описание |
|---|---|
| Разные платформы | Подписка с разных ОС сверх лимита |
| Массовый запрос | 5-10+ устройств запрашивают подписку за 5 минут |
| Необычный трафик | Трафик значительно выше среднего |

### Уверенность и смягчающие факторы

Каждая оценка сопровождается:
- **Уверенность (confidence)** — процент, отражающий надёжность вывода
- **Смягчающие факторы** — автоматические пояснения, почему сигнал может быть ложным: мобильный оператор, одна страна, мало устройств, низкий трафик, все IP домашние

```

│   ├── detect.js            # Движок детекции v6 (13+ сигналов)
│   ├── rules.js             # Движок правил (условия и уведомления)
│   ├── ai-service.js        # ИИ-анализ (OpenAI / Anthropic / Gemini / Custom)
│   ├── ip-check.js          # Проверка VPN/Proxy/Tor (proxycheck.io)
│   └── utils.js             # Общие утилиты идентификации пользователей
├── public/
│   ├── index.html           # SPA (7 вкладок)

└── remnawave-monitor.service.example  # systemd unit
```

### Как это работает

1. **Синхронизация** — сервер каждые 60 секунд забирает данные из Remnawave API: пользователей, активные IP, устройства, историю подписок
2. **Гео-обогащение** — IP обогащаются данными о стране, городе, провайдере через [ipwho.is](https://ipwho.is)
3. **Проверка IP** — проверка на VPN/Proxy/Tor через [proxycheck.io](https://proxycheck.io)
4. **Детекция** — после каждого цикла запускается движок из 13+ сигналов
5. **Правила** — проверяются пользовательские правила, создаются уведомления
6. **Инциденты** — аномалии автоматически создают/обновляют инциденты
7. **Обновления** — клиенты получают данные через SSE в реальном времени

### Безопасность

- API-токен хранится только на сервере — браузер его не получает
- Авторизация через логин/пароль с `HttpOnly` cookie
- Сессии хранятся в SQLite и переживают перезапуск сервера
- Rate limiting: вход — 5 попыток/мин, действия — 30/мин
- Защита от SSRF: прокси пускает только `/api/` пути, только HTTPS
- Заголовки безопасности: X-Content-Type-Options, X-Frame-Options, Referrer-Policy
- Graceful shutdown при остановке сервиса
- Автоматический бэкап базы данных каждую ночь в 03:00

## 📦 Технологии

- **Среда выполнения**: Node.js (без фреймворков, чистый модуль `http`)
- **База данных**: SQLite через [better-sqlite3](https://github.com/WiseLibs/better-sqlite3)
- **Фронтенд**: Vanilla JavaScript SPA
- **Карты**: [Leaflet](https://leafletjs.com/)
- **Шрифты**: [Inter](https://rsms.me/inter/), [JetBrains Mono](https://www.jetbrains.com/lp/mono/)
- **Reverse Proxy**: [Caddy](https://caddyserver.com/) (рекомендуется)
